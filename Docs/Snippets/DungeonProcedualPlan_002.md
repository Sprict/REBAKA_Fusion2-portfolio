フローグラフ自体に指定できるランダム要素

// フローグラフのノード定義例
new FlowNode {
type = RoomType.Combat,
count = new Range(2, 4), // 2〜4部屋をランダムに生成
difficulty = new Range(1, 3), // 難易度1〜3からランダム
heightPreference = HeightPref.Any, // 高さは自由
}

new FlowBranch {
probability = 0.5f, // 50%の確率でこの分岐を生成
nodes = { SecretRoom, TreasureRoom }
}

new FlowNode {
type = RoomType.Boss,
count = 1, // ボスは必ず1部屋（固定）
heightPreference = HeightPref.Highest, // 最も高い場所に配置
distanceFromStart = Distance.Max, // スタートから最も遠く
}

つまり、同じフローグラフでも無限のバリエーションが生成されます。フローグラフは「面白いダンジョンのルール」を定義するもので、「固定のマップ設計図」ではあ
りません。





SGFとWFCは柔軟性の方向が違います。

  SGF vs WFC: 柔軟性の比較

             ゲームデザイン制御
             （面白さ・ペーシング）
                  ↑
                  │
           SGF ★★★★★
                  │
                  │        ←最適なバランス→
                  │
           WFC ★★★
                  │
      ────────────┼─────────────→ 空間的バリエーション
                  │               （有機的・予想外の構造）
           SGF ★★★★
                  │
           WFC ★★★★★
  ┌────────────────────────────────────────────────────────┬─────┬─────┐
  │                      柔軟性の種類                      │ SGF │ WFC │
  ├────────────────────────────────────────────────────────┼─────┼─────┤
  │ ゲームデザイン制御（ペーシング、難易度曲線、Lock-Key） │ ★5  │ ★3  │
  ├────────────────────────────────────────────────────────┼─────┼─────┤
  │ 空間の有機性（予想外の構造、自然な形）                 │ ★4  │ ★5  │
  ├────────────────────────────────────────────────────────┼─────┼─────┤
  │ グローバルルール（「ボスは最遠に」等の全体制約）       │ ★5  │ ★2  │
  ├────────────────────────────────────────────────────────┼─────┼─────┤
  │ ローカルルール（「壁の隣には柱」等の隣接制約）         │ ★3  │ ★5  │
  ├────────────────────────────────────────────────────────┼─────┼─────┤
  │ 立体構造のバリエーション                               │ ★4  │ ★5  │
  └────────────────────────────────────────────────────────┴─────┴─────┘
  核心的な違い

  - SGF = トップダウン設計: デザイナーが「全体の流れ」を決め、アルゴリズムが「詳細」を埋める
    - 長所: 面白さを保証できる
    - 短所: デザイナーの想像を超えるレイアウトは生まれにくい
  - WFC = ボトムアップ生成: タイルの隣接ルールから「全体構造」が創発する
    - 長所: デザイナーが思いつかない有機的な構造が生まれる
    - 短所: 「面白い」かどうかは運次第（後から検証が必要）

  ハイブリッドという選択肢

  実は両方の良いとこ取りが可能です:

  SGF（全体構造）: Start → Combat → Treasure → Boss
    ↓
  各部屋の「内部」をWFCで生成:
    Combat部屋の中の柱・壁・障害物の配置をWFCで有機的に
    Treasure部屋の中のアイテム・装飾をWFCで自然に

  つまり:
  - SGFが「どんな部屋をどの順で配置するか」（マクロ構造）
  - WFCが「各部屋の中をどう飾るか」（ミクロ構造）

  これが最も柔軟かつ品質の高いアプローチです。
