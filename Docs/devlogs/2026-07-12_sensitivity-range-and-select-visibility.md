# 2026-07-12 感度レンジ拡張＋Select表示の入力モード切替

## 問題

1. **感度上限不足**: 右スティック感度Xを上限（3.0）にしても回転が遅すぎた。
2. **Select状態の常時表示**: マウス操作中でも `LeaveSessionButton` 等が常に
   ゲームパッド用の Select 状態になっていた。本作はホバー色と Select 色を
   同色にしているため、「ホバーしていないのに光っているボタン」に見えて
   マウスホバーと混同する。

## なぜこのアプローチか（不採用案も）

### 感度レンジ: スライダー=常用域（0.1〜10.0/0.1刻み）、数値入力=全域（0.01〜10.00/0.01刻み）

- スライダーの範囲を広げすぎると常用域（1前後）の分解能が死ぬ。
  「スライダーは粗く狭く、テキスト入力は細かく広く」は設定UIの定石で、
  極端な値はタイプできる人だけが使えばよい。
- **2026-07-12 追記（レートベース化後）**: 右スティックは `stickToLookDelta` で
  デルタ等価化され、感度 1.0 が旧 8.0 相当になった。テストプレイで上限 10 で十分と判明し、
  `MaximumSensitivity` を 50 → **10** に戻した（スライダー上限と同値）。
- 刻みはゲームパッド左右でも 0.1 になるよう、スライダー内部を「0.1 単位の整数」
  （`wholeNumbers`）に載せ替える（Unity Slider のナビ刻み `(max-min)*0.1` 対策）。
  数値入力由来は 0.01。
- スライダー範囲外の値を数値入力した場合、実値は保持され、
  スライダーのつまみは端に張り付く（`SetValueWithoutNotify` が表示だけクランプ）。
  その状態でスライダーを触ると 10.0 以下に戻る——標準的な挙動として許容。
- 不採用だった「カメラ側の基準速度を上げる案」は、レートベース Processor 導入で
  実質達成済み（1.0x≒快適）。

### Select表示: 「最後の入力種別」でモード切替（`UiInputMode`）

- ルール: Navigate系入力（スティック / D-Pad / WASD / 矢印 / 決定ボタン）で表示、
  ポインタ入力（マウス移動・クリック・ホイール）で非表示。
  同一フレームで両方来たら Navigate 優先。
- 非表示化は `EventSystem.SetSelectedGameObject(null)`。復元は各UIが保持する
  `_lastSelection`（無ければ先頭フォーカス）へ。
  **例外**: `InputField` 編集中は選択解除すると編集フォーカスまで切れるため維持する。
- `UiInputMode` は poll ベースの静的クラス。SettingsMenuController と LobbyMenuUi の
  両方の `Update()` から `Poll()` を呼び、フレーム番号ガードで二重実行を防ぐ。
  action コールバック方式にしなかったのは、各UIが別々の
  `REBAKA_Fusion2`（InputActionAsset インスタンス）を持っており、
  どのインスタンスの Navigate に紐付けるかという所有権の問題が生じるため。
  デバイス直接ポーリングなら所有者がいない。
- 注意点: 生のスティック値を読むため Input System のデッドゾーン処理を通らない。
  自前の閾値（0.25）でドリフト誤検出を防ぐ。
- 不採用: **selectedColor と highlightedColor を別色にする案**。
  混同は解けるが「マウス操作中も常にどこかが光っている」ノイズは残る。
  R.E.P.O 等の実例と同じく「使っていない入力系のカーソルは消す」が正解。

## 仕組み（変更ファイル）

- `InputSettingsData.cs`: `MinimumSensitivity=0.01 / MaximumSensitivity=10`、
  スライダー用の `SliderMinimumSensitivity=0.1 / SliderMaximumSensitivity=10`。
  `Normalize()` は全域でクランプ。
- `SettingsMenuController.cs`: スライダーを 0.1 単位の整数に載せ替え（ゲームパッド
  ナビでも 0.1 刻み）。`Update()` で `UiInputMode.Poll()` → 非表示モード時は選択を
  `_lastSelection` に退避して null 化（InputField 編集中を除く）、表示モード時は復元。
- `UiInputMode.cs`（新規）: 上記のモード判定。
- `LobbyMenuUi.cs`: ロビーの Host/Join にも同じ表示/非表示ルールを適用
  （マウスユーザーが常に Host が光って見える同種の問題を予防）。

## 検証

- コンパイル: エラー0・警告0。EditMode テスト（感度レンジ・スライダー単位変換）パス。
- Play モード: 起動直後は選択なし（マウス想定）→ 矢印キーで HostButton 復元＋Navigate
  移動 → マウス移動で再び非表示、を確認。
- 実機の操作感（ゲームパッド実機・感度）はユーザーがテストプレイで確認。
