# スポーン時自動接地補正（SnapRigToGround）— 浮遊による歩行不能の修正

日付: 2026-06-11 / 関連: 2026-06-10 スナップショット補間検証で発覚

## 問題

Test_Playground でプレイヤーが Space+WASD でしか動かない。

調査で確定した因果チェーン:
1. 静的環境15オブジェクトが未定義レイヤー7にあり、足の groundLayer(=Ground/3) と不一致 → レイヤー3に修正（シーン）
2. レイヤー修正後も、バランス制御がスポーン時の高さを維持するため
   リグ（prefab scale 0.2）の脚長に対しスポーン高さが約0.5m 高く、足が床に届かず浮遊
3. 接地フラグが立たない → Ragdoll/Idle から歩行力が適用されず、ジャンプ中の空中制御のみ有効

検証実験: 重力は正常（テスト球落下）/ 全コライダー底面の最下点=足（何も床に触れていない）/
ワールドジョイントのアンカーを 0.52m 下げるとリグが沈み grounded=True に変化。

## 修正（RagDollPhysics.cs）

`SnapRigToGround()`: 全パーツの最下コライダー底面と Ground レイキャストの差分だけ
リグ全体とワールドジョイントアンカーを平行移動。`EnsureSnappedToGround()` が
UpdatePhysics（権威 tick）から呼ばれ、以下の制御で実行:

- スポーン直後 192 tick（約3秒）は待機 — 姿勢制御の安定前にスナップすると
  足が数cm持ち上がる位置で均衡してしまう（実測: 一発スナップでは右足 4.8cm 浮きで停止）
- その後、足の接触イベント（_isAnyFootGrounded）成立まで毎 tick 補正（最大256 tick）
- 成立したら永久に停止。StateAuthority のみ実行

検証: Host 起動から完全自動で rootY 1.43→0.92、grounded=True、state=Idle を確認。

## 教訓

- アクティブラグドールの「立ち高さ」はスポーン高さに暗黙依存する（バランス制御が
  スポーン姿勢を基準に維持するため）。スケール変更時はスポーン高さの整合が必須
- 物理リグへの一発補正は姿勢制御と干渉する。安定待ち + 達成条件付きリトライが正解
