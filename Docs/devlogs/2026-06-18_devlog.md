## Log

1. PlayerSpawnerにObjectSpawnerの責務を集約した
2. ホストクライアント間でObs_Cubeの位置や回転が同期されるようになったが、なぜかゲーム開始後のObs_Cubeの初期位置がホストクライアント間で異なり、ホストまたはクライアントプレイヤーがObs_Cubeに触れるとクライアント側のObs_Cubeがホスト側と同じ位置にテレポートする問題が発生。
3. 【原因】NetworkRigidbody（非kinematic + 重力あり）は同期に補間バッファを使う。Fusion の補間レンダリングは「直前 tick の状態」と「現在 tick の状態」の2点間を補間します。スポーン直後はバッファに過去データが1点もないため、クライアントがプレハブ保存時のデフォルト位置 (1.5, 0.5, 4) で描画してしまう。「触れた瞬間にテレポート」するのは、接触で StateAuthority が変化し _stateAuthorityChangeErrorCorrectionDelta: 0（即スナップ設定）が発動するためです。これで補間バッファが上書きされ、以降は正常同期します。
Spawned() で TeleportToPosition を呼ぶことで、補間バッファをスポーン位置で即時初期化します。Treasure も同じ構造になる可能性があるため汎用スクリプトとして作ります。

4. クライアント側のプレイヤーをスポーンする前にオブジェクトをすべて現在のホスト側のオブジェクトの位置（クライアントがゲームに途中参加した場合、ホスト側のオブジェクトが既に動かされている可能性がある)にスポーンすることはできないのか?

5. Obs_CubeプレハブのPositionが(0.0, 0.0, 0.0)でなく(1.5, 0.5, 4)のままなのに気づいてポジションリセットしてアセット保存➤そしたらゲーム開始後のホストとクライアントで生成位置が違う問題がなくなった

6. CubeはInterpolateされて予測で動いているのでhostからの位置をそのまま描画しているプレイヤーと違って2tick以上は先行している。そのためプレイヤーがCubeを押すときなどにCubeとプレイヤーの間に無視できない隙間が発生してクライアント側からはプレイヤーはCubeに触れていないのに押しているように見える。その時ホスト側ではちゃんとプレイヤーはCubeに接触して押しているように見える。[プレイ動画](file///D:/ビデオ/REBAKA_Fusion_Test/boxはInterpolateあり、プレイヤーはsnapshot補完.mp4)
