deep-research ワークフローの敵対的検証フェーズは3回試行（Fable 5デフォルト・Fable 5 resume・Sonnet 5明示指定）すべてセッション上限で全滅した。そのため、**Orchestrator（メインモデル）が手動で4体のWorkerサブエージェント（general-purpose・個別WebSearch/WebFetch）にファクトチェックを分担させ、決定寄与度の高いLethal Company / R.E.P.O. / GTFO / DRG Rogue Coreの計21クレームを個別検証した**（2026-07-07実施、詳細は「4体Worker統合検証結果」節）。この4作品の主要数値は「確認」または「一部訂正」まで到達済み。Content Warning / Phasmophobia / Dark and Darker はこの追加検証の対象外のため、引き続き **[※未確認]** として扱う。

## 1. 各作品の進行構造

### Lethal Company（回収ノルマ型の原型）

- **1ラウンド構造**: 会社船（拠点）→ 惑星選択・着陸 → 施設潜入でスクラップ回収 → 船へ持ち帰り離陸。1日=1ラウンド（実時間700秒=11分40秒 ✅検証済）。船は8:00着陸・23:57頃に全員未搭乗でも自動離陸するハードタイマー ✅検証済。ノルマサイクルは4日（0日目含む、実質3日）で、締切日に会社へ売却。[出典 https://lethal-company.fandom.com/wiki/Time, /wiki/Profit_Quota]
- **時間圧**: (a) 惑星ごとの1日サイクル（夜になるほど敵スポーン増・0時ピーク、閾値はmoonのTier依存 ✅検証済）、(b) ノルマ締切という日数単位の明示タイマー、の二層。さらに**moonごとに屋内/屋外/昼行別の固定power level予算があり、スポーン中エンティティのpower合計が上限を超えない容量制御** ✅検証済（例: Titanはindoor 18 / outdoor 7 / daytime 20）——REBAKAのコスト容量式と同型。締切残日数もスポーン圧を変調するが単調増加ではなくU字型（残り3日が最小）⚠️。ノルマ未達成はクルー全員宇宙放出＝ゲームオーバー。
- **メタ進行**: ノルマは達成のたびに二次関数的に増加（初回130固定 ✅検証済、逓増は達成回数の2乗に比例。係数の正確な式は一次原文未到達のため[※未確認]）。早期達成でオーバータイムボーナス。金で装備・船内家具を購入（家具のluck値がノルマ増加を緩和するという報告もある[※未確認]）。恒久解放はほぼ無く「どこまで生き延びたか」のスコアアタック的ループ。
- **死とペナルティ**: 死亡者は装備・所持スクラップを喪失。ラウンド中の復活は無く、船が軌道に上がる（＝ラウンド終了）と全員復帰。遺体を船へ回収すると金銭ペナルティが軽減。[※未確認: 出典 https://dotesports.com/indies/news/how-to-respawn-in-lethal-company]

### R.E.P.O.（物理運搬×ラン内成長 — 最も近い参考作）

- **1ラウンド構造**: レベルへ潜入 → 物理オブジェクトの貴重品を壊さず運搬・回収 → 抽出ポイントへ納品 → 次レベルへ。物理運搬そのものがコア動詞。
- **時間圧**: 明示タイマーではなく、騒音・破損・敵の徘徊という環境リスクが主。貴重品は壊れると価値が下がるため「急ぐほど損」の逆向きの圧が働き、丁寧さと速度のトレードオフになる。[※推測を含む: 大枠は https://www.switchbladegaming.com/co-op-games/repo/vs-lethal-company/ より]
- **メタ進行**: レベル間の Service Station で通貨（SURPLUS）を消費し、最大HP等をそのラン内で恒久強化する「ラン内成長型」✅検証済。利益は次ラウンドへ持ち越し。ノルマ制もあり、抽出ポイント数はレベルで逐次増加（Lv1=1 / Lv2-3=2 / Lv4-5=3 / Lv6-19=4 / Lv20+=5、前の1つを使い切ると次が出現）✅検証済。超過分は「Tax return」として次の抽出へ持ち越せる。
- **死とペナルティ**: 死亡者は頭部（物理オブジェクト）となり、仲間が抽出ポイントへ運搬＋ノルマ達成の両条件で抽出発動時に復活 ✅検証済。**蘇生後HPは1のみ**（船内で最大25まで自動回復）。ラウンド終了時は生死問わず全員25HPで自動蘇生する救済も別途あり。死体すら物理運搬対象という設計が特徴。[出典 https://itemlevel.net/how-to-revive-your-friends-in-repo/ ほか]

### Content Warning（1日1ダイブ×映像素材という「持ち帰り」の別解）

- **1ラウンド構造**: 準備（顔カスタマイズ・装備購入）→ ダイビングベルで旧世界（Old World）へ降下 → 怖い映像を撮影 → 生還 → 映像をSpöökTubeにアップロードし再生数=金を稼ぐ。1ランは3日間で構成され、3日で十分な再生数を稼げばキャリア続行 ✅検証済（3-0票）。持ち帰るのが「物」ではなく「記録メディア」なのが独自。[出典 https://contentwarning.wiki.gg/wiki/Main_Page]
- **時間圧**: 酸素残量が実質のラウンドタイマー。カメラのバッテリー/フィルム容量も撮影時間を制限する。[※未確認: 「酸素/バッテリー消耗資源が時間圧」というクレームは検証で0-3反証された（出典のwikiトップページに該当記述が無いため）。酸素メカニクス自体は学習知識と整合するが、一次出典未到達]
- **メタ進行**: 動画の広告収入で装備アップグレード ✅検証済（3-0票）。
- **死とペナルティ**: 死んでも次の日に復活するが、死亡者のカメラを回収しないと映像（＝収益）を失う。「遺体でなくカメラを回収する」インセンティブ設計。[※学習知識ベース、要裏取り]

### GTFO（超高難度・セッション単位の遠征型）

- **1ラウンド構造**: Rundown（期間限定ミッション群）から遠征を選択 → 装備ロードアウトを事前選択 → 潜入し目標達成 → 抽出。1遠征が長く（1〜2時間級）、失敗＝最初から。
- **時間圧**: 明示タイマーは局所（アラームイベント等）のみ。主圧は資源枯渇（弾薬・医療の有限性）と、警報による敵ウェーブ。アラームは多くの場合「ランピング」性質を持ち、**持続時間が長いほどウェーブのスポーン数・頻度が増加** ✅検証済（ただし全アラーム共通の法則ではなくレベルデザイン側の調整パラメータ）。サージアラームは**キル即リスポーンで上限まで補充し続ける** ✅検証済。[出典 https://gtfo.wiki.gg/wiki/Alarms]
- **メタ進行**: 恒久成長ほぼ無し。進行は「どの遠征をクリアしたか」のみ。純スキルベース。
- **死とペナルティ**: チェックポイント制がRundown 6.0（正式リリース時）で導入され、全滅時はロビーか最終チェックポイントへ。チェックポイント保存は4人全員のバイオスキャンが必要 ✅検証済。[出典 https://gtfo.wiki.gg/wiki/Checkpoint]

### Deep Rock Galactic（ミッション選択型・王道メタ進行）+ Rogue Core

- **1ラウンド構造**: 宇宙船ハブでミッション選択 → 降下 → 目標（採掘・回収等）達成 → Molly（運搬ロボ）に納品 → 脱出ポッドへの帰還ダッシュ。脱出フェーズが強制的に時間制限化される（ポッド発射までのカウントダウン）。
- **時間圧**: ミッション中は基本タイマー無し・ウェーブ襲撃型。終了条件成立後の脱出カウントダウンで最後だけ急かす構造。
- **メタ進行**: 恒久成長が最も厚い参考作。金・素材でクラス別装備アンロック・オーバークロック・コスメティック。失敗してもXPと集めた資源の一部は残る「失敗しても前進」設計。
- **死とペナルティ**: 味方蘇生が自由（ダウン→起こす）。全滅でミッション失敗だが報酬の一部は保持。
- **Rogue Core（ローグライトスピンオフ）**: ラン中はExpeniteをR.E.P.D.装置に預けてアップグレード取得（初回120、逓増は+10刻み→150以降+15刻み→300以降さらに加速の3段階）✅検証済、co-op向けターン制ドラフト・共有プール。ラン終了で武器・アップグレードは全リセット、持ち越しはIntel（進行）・Enhancements（恒久バフ）・コスメティックの3種のみ ✅検証済。Intel Taskはミッション失敗でも部分進行が残る（一部例外あり）。[出典 https://deeprockgalactic.wiki.gg/wiki/Rogue_Core:Upgrades, https://www.keengamer.com/articles/guides/deep-rock-galactic-rogue-core-what-carries-over-between-runs/]

### Phasmophobia（調査型・非戦闘）

- **1ラウンド構造**: ロビーで装備積み込み → 現場調査 → 証拠3種からゴースト特定 → トラックへ帰還・撤収。「回収」対象が物でなく情報。
- **時間圧**: セットアップ時間（猶予）終了後にハント（襲撃）が解禁され、平常心（Sanity）が下がるほどハント頻度が上がる。Sanityが実質の環境悪化タイマー。
- **メタ進行**: 金で装備購入（持ち込み装備は消耗リスク資産）、レベルで装備ティア解放。
- **死とペナルティ**: 死亡者は装備をその場に落とし観測者化（生存者と会話不可）。タロット『The High Priestess』やMonkey Pawでラウンド中復活の可能性あり。死亡したままラウンド終了だと報酬半減＋難易度倍率無効、装備喪失（保険: Amateur 50% / Intermediate 25% / Professional以上 0%返却）。[※未確認: 出典 https://phasmophobia.fandom.com/wiki/Death]

### Dark and Darker（PvPvE抽出型 — 「抽出」の緊張感の参考）

- **1ラウンド構造**: 装備を持ち込んでダンジョンへ → 宝回収・PvPvE → 縮小するセーフゾーン内で脱出ポータル/シャフトを確保して抽出。
- **時間圧**: 縮小リング（Dark Swarm）による強制的な空間圧＝実質タイマー。脱出手段が有限なため終盤にプレイヤー間の競合が発生。
- **メタ進行**: 持ち出した装備・金がそのまま資産。トレード・スタッシュ蓄積。
- **死とペナルティ**: 死＝持ち込み装備含む全ロスト（フルルートされる）。抽出型の最も苛烈なペナルティ。
- [※学習知識ベース: ワークフロー出典に含まれず。仕様変動が激しいタイトルなので要最新確認]

## 3. 横断的な設計パターン

1. **時間圧は「明示タイマー」より「状態悪化」が主流**。Lethal Companyの夜、PhasmophobiaのSanity、GTFOの資源枯渇、Dark and Darkerのリング——どれも数字のカウントダウンではなく世界の状態変化として提示され、ホラーの没入を壊さない。明示カウントダウンを使うのはDRGの脱出フェーズのように「終盤だけ」が定石。
2. **死のペナルティは「回収可能な喪失」にするとco-opの動詞が増える**。Lethal Companyの遺体回収、R.E.P.O.の頭部運搬、Content Warningのカメラ回収は、死をゲームオーバーではなく「仲間の新しいタスク」に変換している。
3. **ノルマ逓増は緩やかな敗北タイマー**。Lethal Companyのループが強いのは「いつかは必ず負ける」二次関数ノルマで1セッションに自然な終わりを作っているから。[※未確認の計算式に基づく解釈]
4. **メタ進行の厚さはセッションの重さと反比例**しがち。GTFO（重い1ラン・メタ無し）⇔DRG（軽い1ラン・厚いメタ）。ラン内成長型（R.E.P.O. / Rogue Core）はその中間で、1ランを複数レベルの連続にすることで成長の快感をラン内に閉じ込める。

## 4. REBAKA への推奨ラウンド構造

前提: 全部物理のアクティブラグドール運搬・4人co-op・エネミーは**コスト容量式リスポーン（時間経過で容量増）**。

### 推奨: 「容量式スポーンを時間圧の主役に据えた R.E.P.O.×Lethal Company ハイブリッド」

**(1) 1ラウンド構造**
- 拠点（安全圏）で準備 → マップへ潜入 → 宝物を物理運搬で抽出ポイントへ納品 → 全員が脱出地点に集合して離脱、でラウンド終了。
- 終了条件は「任意離脱 or 全滅」。**離脱はいつでも選べる**ようにする（Lethal Company型）。理由: コスト容量が時間で増える設計は「粘るほど地獄」を自動生成するので、"もう1個だけ取るか、今帰るか" のリスク判断そのものをラウンドの主ドラマにできる。明示タイマーは不要。

**(2) 時間圧**
- **主軸は自作のコスト容量そのもの**。容量増加をプレイヤーに数字で見せず、環境演出（照明悪化・音・敵の密度）で体感させる（横断パターン1に準拠）。
- 補助として、納品や大騒音（物理衝突！）で容量に一時ボーナスを与えると「物理で暴れるほど危険」という自作の物理コアと時間圧が接続する。R.E.P.O.の騒音リスクの容量式翻訳であり、これは他作に無い差別化ポイント。

**(3) メタ進行**
- **ノルマ逓増型（Lethal Company式）を採用**し、3〜4ラウンドで1締切。既に採用済みの「コスト容量式リスポーン」と対を成す「収入側の逓増圧」になり、セッションに自然な終わりを作る。
- アップグレードは**ラン内成長型（R.E.P.O./Rogue Core式）を第一段階として推奨**。恒久メタ（DRG式）はコンテンツ量が要るためポートフォリオ規模では過剰。持ち越しは記録（最高到達ノルマ等）だけで十分。

**(4) 死と復帰**
- **R.E.P.O.式の「死体＝物理運搬対象」を強く推奨**。死亡者はラグドール（まさに自作のコア技術）となり、仲間が抽出ポイントまで物理運搬すれば次ラウンド（または納品時点）で復活。宝物とまったく同じ運搬メカニクスで死体も運べる——「全部物理」のコンセプトが死のルールにまで一貫し、説明可能性も高い。
- 死亡中は観戦（Phasmophobia式観測者）。全滅でラウンド失敗、回収済み宝は喪失（未納品分のみ）。

### 不採用案と理由
- **明示カウントダウン（Content Warningの酸素式）**: コスト容量式と役割が被り、二重タイマーは判断をノイズ化する。
- **GTFO式の重い1ラン＋メタ無し**: 高難度前提のプレイヤー層向けで、物理コメディ要素のあるラグドールゲーとトーンが合わない。
- **Dark and Darker式の全ロスト**: PvP資産経済があって初めて成立する苛烈さ。PvE co-opでは離脱率を上げるだけ。

### 検証で得た設計上の追加知見

1. **Lethal Companyのpower level予算制は、REBAKAのコスト容量式とほぼ同型の実装が既に商業的に成功している証拠**。差別化点は「予算が時間で増える」動的性（LCは moon ごとに固定）に置ける。
2. **LCの締切連動スポーンがU字型**なのは示唆的: 「締切直後の緩和」はプレイヤーの回復期を意図的に作る設計と読める。REBAKAでもラウンド序盤に容量を意図的に絞る「呼吸」の設計に応用できる。
3. **R.E.P.O.の蘇生はHP1復帰**: 復活を「無料」にせず、復活直後の脆弱性でリスクを残す。REBAKAで死体運搬復活を採用する場合も同様のコスト設計を推奨。
4. **GTFOのランピングは"often"＝レベルごとの調整パラメータ**: 増加曲線をグローバル定数ではなくマップ/イベント単位で調整可能にしておくと、後からの難易度調整が効く。
