## 🎯 ドキュメントの使い方

### Phase 2開始時（2/1）
1. **SPEC.md**を読み直し、今週のタスクを確認
2. **TECHNICAL_DESIGN.md**を開き、[TODO]を順次埋めていく
3. **devlogs/**に日々の進捗を記録

### Phase 2終了時（2/14）
1. **TECHNICAL_DESIGN.md**の全[TODO]が埋まっていることを確認
2. TECHNICAL_DESIGN.mdの「技術理解Q&A」セクションを復習し、技術的な質問に答えられる状態にする

## 🔄 ドキュメント更新ルール

共通のブランチ運用とドキュメント運用ルールは、リポジトリ直下の `AGENTS.md` を参照すること。
Codex、Antigravity、Claude Code を含む複数エージェントで同じ基準を使う。

### 更新頻度
- **SPEC.md**: 大きな方針変更があった時のみ
- **TECHNICAL_DESIGN.md**: Phase 2期間中、毎日少しずつ
- **devlogs/**: 毎日の作業終了時

### バージョン管理
- 大きな変更時は、古いバージョンをarchive/に移動
- 日付をファイル名に含めて区別

---

## 📌 重要な原則

1. **SPEC.mdが最上位の仕様書**
   - 他のドキュメントと矛盾する場合、SPEC.mdが優先

2. **TECHNICAL_DESIGN.mdが技術理解の核**
   - このドキュメントが完成すれば、Phase 2は成功

3. **正直な記録**
   - AIとの協働、理解が不十分な部分を隠さず記録する
   - 誠実さがポートフォリオの価値になる
