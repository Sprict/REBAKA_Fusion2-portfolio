# Docs の読み方

> **最終更新:** 2026-07-13

採用選考・技術レビューで読んでいただく想定の案内です。時間に応じて次の順をおすすめします。

## 3分コース

1. リポジトリ直下の [`README.md`](../README.md) — 概要・動画・技術ハイライト
2. [`ARCHITECTURE_OVERVIEW.md`](ARCHITECTURE_OVERVIEW.md) — ネットワーク物理同期の全体像を1枚で

## 15分コース（上に加えて）

3. おすすめ devlog 3本（ルート README 末尾に案内があります）— 問題 → 不採用案 → 採用理由 → 検証、の実記録
4. [`ARCHITECTURE_FAILURE_MODES.md`](ARCHITECTURE_FAILURE_MODES.md) — 「どう壊れるか」の整理

## 各ディレクトリ

| 場所 | 内容 |
|---|---|
| `devlogs/` | 開発の時系列記録（問題・原因・設計判断・検証） |
| `TECHNICAL_DESIGN.md` | 技術設計の詳細（長文。上記を読んでから） |

本リポジトリは開発リポジトリからの抜粋公開版です。ドキュメントの整理・改稿は継続中で、現在の内容と一部食い違う古い記述が残っている場合があります。
