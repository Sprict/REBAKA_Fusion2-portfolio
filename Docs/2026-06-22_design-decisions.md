## 1. 笑いの3つのエンジン（確定）

今日の会話で「5秒のクリップ」を起点に、あなたが自然に繰り返し使った笑いのパターンが3つに収束した。
設計判断を下すときは、この3つのどれかに引っかかるかを検証基準にする。

| エンジン | 説明 | 実例 |
|---|---|---|
| **① 大事な物を雑な鈍器に** | 命懸けで運んでいる一番大事なものを、その辺の武器みたいに雑に使い倒す | 宝で仲間の死体を殴って昇天阻止 / モンスターに宝を投げつける / 宝をトラップの囮・盾にする |
| **② 力みすぎて世界が壊れる** | 助けようとして力を入れすぎると、世界の方が崩れる | トラばさみに挟まった足ごと引きずったら地面の石板ごとぶっこ抜ける |
| **③ 不謹慎な心** | 事故でなく「性格の悪さ」から生まれる笑い | 仲間をトラップに見捨てて先に行くフリをする |

### 世界観のトーン（確定）

> **世界は真顔。アホなのはプレイヤー。**

洞窟・道具・宝・トラップは全部「ちゃんとした重い物理オブジェクト」。
ネタ小道具（ピコピコハンマー等）は要らない。**笑いはプレイヤーの行為から生まれる。**
世界がボケないからこそ、プレイヤーのアホが映える。参考: Lethal Company / R.E.P.O.

## 2. 宝の設計（確定）

### 2つの軸

宝は**価値**と**耐久力**の2軸を持つ消耗品。

```
高価値 × 低耐久力  →  一撃で壊れるが高額。丁寧に運ばないといけない。
低価値 × 高耐久力  →  安いが頑丈。囮・盾・殴る武器として雑に使いやすい。
```

### R.E.P.O.との根本差

| | R.E.P.O. | このゲーム |
|---|---|---|
| 宝の役割 | 守るべき脆いもの | 使い倒していい消耗品 |
| プレイヤーと宝の関係 | 宝に奉仕する | 宝を支配する |
| コアの問い | 「壊すな」 | **「壊してでも帰れ」** |

### 宝の用途（確定）

1. **ゴール**: 持ち帰ってスコアになる（本来の目的）
2. **武器**: モンスターに投げつけて追い払う
3. **救出道具**: 打撃で天使の昇天を即座に阻止する（耐久力を消費）
4. **トラップ対策**: 重みに反応するトラップへの囮 / 矢の盾

## 3. 救出システム（確定）

死亡 → 一定時間後に天使2体が昇天させようとする → 仲間が阻止できれば HP1 で復活。

### 2つの阻止手段

| 手段 | 速さ | コスト | 笑いのエンジン |
|---|---|---|---|
| **素手で掴んで引っ張る** | 遅い（時間がかかる） | 自分も浮くリスク → 人間の鎖 | ① 連帯責任・巻き込まれ |
| **打撃アイテムで殴る** | 即座 | アイテム耐久を消費（宝も可） | ① 大事な物を鈍器に |

**武器はモンスター戦闘とも兼用** → 「今ここで使うか」のリソース判断が常に走る。

### 天使の仕様（確定）

- 上昇は**微加速**（じわじわ上がる）
- **1人で止まる**（誰か1人が掴んで引っ張れば昇天を止められる）
- 複数人が同時死亡しても**挙動は変わらない**。各死体の昇天は**死亡タイミングから個別に計算**する
- 昇天が完了すると**光とともにフェードアウト**して消える

### ゴットレイゾーン方式（確定・救出ロジックの核）

死＝高さではなく**ゾーン滞在時間**で判定する。

```
死亡 → 死体の上にゴットレイ（光の柱）のゾーンが発生
     → 死体がゾーン内に一定時間とどまると昇天（＝本当の死・復活不能）
     → 救出 = 死体をゾーンの「外」へ引きずり出すこと
```

この方式の利点:

- **天井の高さに依存しない**。低い天井で死体があまり上昇しなくても、ゾーンから出せれば救出成立。
- マップ自動生成で天井高がまちまちでも、救出ロジックが破綻しない。
- 「昇天を止める（掴む・殴る）」と「ゾーンから出す（引きずる）」の2段構えになり、救出の駆け引きが生まれる。

## 4. 死後システム（確定）

### 幽霊の仕様

- 通常のアクティブラグドールとして**地面を歩く**、地形の影響を受ける
- モンスター・トラップには**反応されない**（無敵）
- **ヘッドライトなし** → 暗所は見えない（生存者のヘッドライト周辺のみ視界あり）
- 生存者からは**見えない**（声も届かない）
- 死者本人には**天使の輪が付いた半透明の自キャラ**として見える
- 死者同士は**声が聞こえる**（あの世のパーティーが賑やかになる）
- **限定エネルギー制で物を弱く動かせる**（協力・いたずら自由）
- **テレポートで生存者を順繰りに巡回**できる（迷子防止。任意選択ではなく順番）
- **生き返れない**（ランが終わるまで幽霊のまま）

### 幽霊の2つの楽しみ方

幽霊は「サバイバルゲーム」を抜けて「観察と介入の神ゲー」に移行する。

| プレイスタイル | 行動 | 動機 |
|---|---|---|
| **協力型** | 斥候・物を動かして誘導・トラップを先に起動 | ランの成功に貢献したい |
| **いたずら型** | ちょっとした妨害・生存者を驚かす | 暇つぶし・自分の干渉で仲間が騒ぐのを見たい |

設計はどちらかを強制しない。**合理的な幽霊は協力する。そうじゃないプレイヤーには暇つぶしの選択肢がある。**

### デザイン意図

「**守護霊コメディ**」: 生存者は「宝が勝手にズレた、ラッキー」としか思わない。
その裏で死んだ仲間が半透明で歯を食いしばって必死に押している。

「**限定的な神視点**」: ヘッドライトがないため完全な神視点ではない。
生存者に追いつかないと暗くて見えない。テレポートも順番巡回のみ。
適度なコストで情報が強すぎない設計になっている。

「**死のコスト維持**」: 幽霊になることは「損」。
- 失うもの: サバイバルの緊張・恐怖・達成感、生存者との会話、宝を運ぶ主体性
- 得るもの: 無敵、神視点、限定介入権
**「死んだ方が楽」にならない**ように、サバイバルの醍醐味（怖い・焦る・笑える）は生存者専用。

### 確定ルール

- 幽霊は生存者に情報を渡せない → **死人に口なし**（資産として維持）
- 死者は生存者の主要な敵にならない → **裏切りレイヤーなし**（いたずらは許容）
- FF（フレンドリーファイア）は**維持**。FFは「事故」のためにあり「敵対」のためではない。

## 5. マップ構造（確定）

### 基本構造

- **ランダム生成**（Lethal Company式）
- **入口 = 出口**（最上層）
- **深いほど良い宝・強いトラップ**
- プレイヤーが自分でリスクを選んで奥へ進む

### なぜ「最下層スタート」を却下したか

最下層スタートだと「奥に進む/進まない」という選択肢がプレイヤーに存在しない。
**自分で選んだリスクが面白さの源**なので、入口スタートが正しい。

### マップ形式

洞窟生成が難しければ**石造りのダンジョン（四角い部屋の組み合わせ）**でも可。
道を覚えて帰れるかが緊張の一要素なので、迷いやすさは維持する。

## 6. アクティブラグドールの差別化（確定）

### R.E.P.O.との根本差

> **R.E.P.O.「物が物理オブジェクト。プレイヤーはカーソル」**
> **このゲーム「プレイヤー自身が物理オブジェクト。物を持つと体が引っ張られる」**

### 地形がプレイヤーの体を常時いじる

トラップ・モンスターは「イベント的」な外力。
**地形は常時の外力。** これがアクティブラグドールを常に活かし続ける舞台装置になる。

候補（絞り込み済み・実装優先度順）:

| 地形 | 効果 | 状況 |
|---|---|---|
| **氷・油** | 摩擦低下 → 滑る。重い宝を持つと加速度的に崩れる | 採用方向（摩擦変化系の本命） |
| **蜘蛛の巣** | 絡まって動けない → 拘束系。仲間が引っ張って救出 | 採用方向 |
| 水流・水たまり | 外力 → 体が押される、抵抗には踏ん張る | 候補（捨てる理由はないが、エフェクト＋自動生成との両立で実装コスト懸念） |
| 樹液 | 粘着 → 減速 | 保留（「地面に広く広がる樹液」は自然さに疑問。要検討） |
| 崩れる床・不安定な足場 | 形が変わる → 予測できない | 候補（実装方法の目途が未確定） |

> 摩擦変化系（氷・油）と拘束系（蜘蛛の巣）の2系統を軸にする。
> どれを最終採用するかは、エフェクト制作コストと自動マップ生成との両立を見て決める。

### 行きと帰りで別のゲームになる（重要）

```
行き（手ぶら）: 素早く動ける、トラップを軽やかに避けられる
帰り（宝を抱えてる）: 宝の重さが体に乗ってくる、同じトラップが別の難しさになる
```

これはR.E.P.O.には絶対できない差別化。
R.E.P.O.は「物が重い」ゲーム。このゲームは「**物の重さが自分の体に乗ってくる**」ゲーム。

## 11. 脅威設計の原則（確定）

> **位置づけ**: これはこのゲームの**背骨になるフレームワーク**。トラップ・モンスター・環境の脅威を
> 設計するとき、毎回ここに照らして「フェアか／理不尽か」を判定する。
> 注記: 以下はゲームデザイン論であり物理法則のような「正解」はない。確立した議論（フェアネス論・テレグラフ論）と推論が混ざる。`[※理論]` `[※推測]` は原則として扱う。

### 発端：作者の2軸モデルには「悪い設計を見逃す」穴があった

作者は当初、脅威のフェアネスを **「予測可能/不可能 × 回避可能/不可能」の2軸**で分類しようとした。
根拠は「Lethal Company は対策・適切な判断で生存可能だから R.E.P.O. より面白い」という実感。

しかしこの2軸で「**見えない死者が、生者の通り道に致命的なトラップを不意に滑り込ませる**」案を採点すると:

> 「死者がトラップを動かせると知っていれば警戒して回避できる」→ **回避可能・予測可能 → 合格**

直感では「理不尽」と却下した案が、**モデル上は通ってしまう**。
モデルと直感がズレるのは、**フェアネスを決める本当の軸が2軸に入っていない**から。そこで6軸へ拡張した。

### 6つの判定軸（2軸 ＋ 追加4軸）

| 軸 | 問い | 重要性 |
|---|---|---|
| **① 予測可能性** | 事前に危険を読めたか（型の予告・パターン学習） | 中 |
| **② 回避可能性** | プレイヤーの行動でその場の結果を変えられるか | 中 |
| **③ 事後可読性（legibility）** | 死んだ「後」に、なぜ死んだか再構築・学習できるか | **最重要** |
| **④ 帰属（attribution）** | 死因が「自分の欲／仲間／純粋な運」のどれに帰属するか | **最重要** |
| **⑤ 反応窓 vs 操作精度** | 回避に要るのは「時間（反応窓）」か「高い操作精度」か | 高 |
| **⑥ 罰の比例性** | 結果（ダメージ・死）が引き金のミスの大きさに見合うか | 高 |

各軸の要点:

- **③可読性 ≠ ①予測可能性**。予測可能性は「事前に読めたか」、可読性は「事後に学べるか」。**学習を生むのは可読性**。予測不能でも可読なら「次はこうする」になる（例: 圧力板を踏んだと分かる）。見えない死者のトラップは可読性ゼロ＝理不尽。
- **④帰属**: 自分や仲間に帰属する死は、**メカニクス的に回避不能でも"フェア"に感じる**。このゲームの笑い（自業自得の死・見殺し・FF事故）は全部この帰属の上に乗っている。見えない死者のトラップは帰属先が不在＝犯人が見えない＝理不尽の正体。
- **⑤反応窓 vs 操作精度**: 予測・回避可能でも反応に0.2秒しかなければ理不尽。**アクティブラグドールは意図的に操作が緩い**ため、「素早く正確に避ける」前提の脅威は操作系と構造的に喧嘩する。CoilHead系（びっくり＋反射神経）を切ったのはこの軸の無意識的適用＝正しい判断。
- **⑥比例性**: 「即死の可否」でなく「ミスの大きさと罰の大きさが釣り合うか」。LCのタレットが即死でも納得なのは、引き金が「明らかに射線に入った」という**大きく分かりやすいミス**だから。

**③可読性 と ④帰属 が最重要。** この2つこそ、このゲームで最も効く軸。

### なぜ「回避可能性」が主軸にならないか（最重要の転換）

このゲームの最高の笑い＝創発的カオス（仲間に突き飛ばされてトラップに落ちる／氷で滑って穴に）は、
**被害者視点では「その場で予測不能・回避不能」**。
「全ての死は回避可能であるべき」を厳格適用すると、**自分の笑いそのものを禁止することになる**。

```
「宝を持ちすぎて崩れた」     → その場では回避不可でも「自分が貪欲だったから」で笑える
「仲間に突き飛ばされて落ちた」 → 回避不可でも「仲間のせい（社会的帰属）」で笑える
「見えない死者にトラップを動かされた」→ 回避不可 + 帰属先が見えない = ただの不満
```

→ **このゲームのフェアネスは「回避可能性」でなく「帰属」から来るべき。** これが今日一番大事な転換。

### Q2: 「回避可能な即死トラップ」はこのゲームを殺しかねない

回避可能であっても、即死トラップ自体に固有の害が3つ:

1. **慎重プレイのメタを生む**。即死が各所にあると最適戦略は「ゆっくり一歩ずつ」。
   だがこのゲームの笑いのエンジンは「怖い→焦る→急ぐ→体がよろける→間抜けな死」。
   **急がせないと笑いが出ない**ため、即死トラップはコメディ設計と利益相反する。
2. **救出ドラマの窓を消す**。天使綱引き・幽霊の助けは全部「脅威と死の"間"の時間」を必要とする。
   即死はその間をスキップ＝自分で作った救出システムを無効化する。
3. **ラグドールの緩さと最悪の相性**（⑤）。

→ [※推測] **トラップは基本「即死」でなく「拘束・負傷・足止め」にすべき**。トラばさみ（拘束）はまさにこれ。
即死は「大きく明白で自業自得なミス」への稀な句読点に留める。

### Q3: Lethal Company の成功理由は「回避可能だから」ではない

作者の「対策可能だから面白い」という一般化は要注意。[※推測] LCのフェアネスの根はもっと下にある4つ:

1. **ルールの一貫性＝学習可能性**。Bracken=「見すぎると襲う／放置なら襲わない」、CoilHead=「見れば止まる」。**一貫した致命ルールはフェア**。回避可能性はこの一貫性の"結果"であって原因ではない。
2. **テル（tell）への投資**。足音・心音・視覚的予兆。フェアネスは抽象的な「回避可能性」でなく具体的なテルに宿る。
3. **社会的・自己帰属**。死の多くは「誰かがドアを開けた」「欲張って奥に行った」に帰属＝理不尽でなく笑い／教訓になる（④）。
4. **情報の不完全性＝認識的恐怖**。見えない・聞こえるだけ・何が湧いたか分からない。**恐怖はメカニクスでなく"知らないこと"から来る**。LCの死の多くは「その場で回避可能」でなく「**何度も遊んで得た知識で回避可能**」。

→ 「LCの方が面白い」という**個人的選好を"対策可能だから"という普遍法則に一般化**しかけていた点が危険。
実際はLCの強さは情報設計と帰属設計で、対策可能性はその副産物。

### Q4: この考えを適用したときの失敗パターン

- **失敗1: フェアにしすぎて恐怖が死ぬ**。全脅威が予測可能＋回避可能＋テル付きなら驚き＝恐怖が消える。**解けたパズルはホラーじゃない**。恐怖には不確実性が要る。
- **失敗2（最重要）: 自分のコメディと矛盾する**。突き飛ばし死・滑り落ち死は被害者視点で「その場で予測不能・回避不能」。「全ての死は回避可能であるべき」を厳格適用すると**笑いそのものを禁止**する。解決は回避可能性でなく**帰属**で許す。
- **失敗3: テルの飽和**。全部に予兆を付けると画面と音が警告だらけ→ノイズ→無視される。テル予算は有限。

### 6つの設計ルール

1. **種類は既知、インスタンスは未知**。「タレットが存在し、こういう音がする」は学ばせ、「この部屋にあるか」は隠す。恐怖はインスタンスの不確実性から、フェアネスは種類の既知性から。
2. **帰属を最優先**。死を必ず「誰か（自分の欲・仲間・自分の滑り）」に辿れるようにする。**回避可能性より帰属可能性**。これが笑いと一致する。
3. **即死より拘束・負傷・足止め**。救出と笑いの窓を残す。即死は「大きく明白で自業自得」な稀な句読点だけ。
4. **事後可読性を保証する**。死んだ本人・観戦者・幽霊が死因を再構築できる。デスカム／幽霊視点が効く。
5. **ルールに例外を作らない**。一貫した致命ルールはフェア。**これが「見えない死者のトラップ」を裁く軸**——回避可能でも、ルールの一貫性と可読性を破るから不可。全トラップ・宝・アイテムは物理オブジェクトとし、幽霊も生存者も同じルールで物を動かす（特権的な見えない手を作らない）。
6. **慎重を罰する圧力を別に置く**。時間／欲（ノルマ）の圧力で「立ち止まりも遅い死」にする。さもないと「回避可能＝ゆっくり行けば安全＝退屈」になる。これが急がせ、カオスと笑いを生む。

### 種明かし：6軸なら、あの案を正しく裁ける

「見えない死者のトラップ移動」を6軸で再採点:

| 軸 | 判定 |
|---|---|
| ② 回避可能性 | △（一応可）← **作者の2軸はここで合格させた** |
| ③ 事後可読性 | ✗（なぜ死んだか分からない） |
| ④ 帰属 | ✗（犯人が見えない） |
| ⑤ ルール一貫性 | ✗（見えない手という例外） |

→ 回避可能性で見ると通るが、**可読性・帰属・一貫性で見ると三重に不可**。
作者の直感（理不尽だ）は正しく、足りなかったのは**直感を言語化する軸**だった。これがモデル拡張の実益。
