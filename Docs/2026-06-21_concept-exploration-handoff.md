## 1. 固定制約（不変・絶対）

- プレイヤーは **アクティブラグドール**。手触りは **Gang Beasts と Human Fall Flat の中間〜やや Gang Beasts 寄り**。操作自体はストレスにしない（フニャフニャだが制御は効く）。
- **フレンドリーファイアあり** の **協力(CO-OP)** ゲーム。**2〜4人**想定（コードは `maxPlayers=4`）。
- 技術: Unity + Photon Fusion 2。操作基盤は 移動 / ジャンプ / 前方へ腕を伸ばして掴む(grab) / 腕を突き出す(thrust) / 掴み中に突き出してタイミングよく離す=投げる(throw)。複数人で重い物を支える **質量シェア運搬**は実装済み（`Treasure.cs` / `TreasureGrabRegistry.cs`）。

## 2. 作者の嗜好（面白さの方向）

- 物理が「協力の文法」になっていること（押し合い・支え合い・巻き込み事故・**連帯責任**）。
- **はたから見て面白い = 5秒で笑えるクリップ的瞬間**が、作り込んだ演出でなく**物理の必然**から生まれること。
  - ただし「クリップ映え」は**北極星でなく自己検証基準**。クリップ映像と実プレイ体験が乖離したらユーザーは離れる、という考え。
- **緊張感と「仲間といる安心・ワイワイ感」の混在**。死すらネタになる。
- 笑いの源は「自分の身体が言うことを聞かない（操作難）」ではなく **「繋がっているせいで巻き込まれる（共有された物理の連動）」**。← Gang Beasts 寄りの素直な操作と整合。

### 確定した見せ場: 「天使昇天 × 綱引き救出」（SPEC §3.3 の改良案）

- プレイヤーが死ぬ → 操作不能でラグドール脱力、地面に倒れる。
- 一定時間後、**光とともに天使が降りて死体を上方へ連れ去ろうとする**（フランダースの犬・パトラッシュの昇天オマージュ）。
- 仲間が死体を掴んで**上昇を阻止**すると **HP1で復活**。失敗で連れ去られ消失（=行方不明）。
- 救出は **「天使の上昇力 vs 仲間の体重・グリップ」の物理綱引き**。足が浮きバタつく姿が観戦的に面白い。
- **なぜ強いか**: ①制限時間が diegetic（天使の上昇＝可視タイマー、UI不要）②救出が物理綱引き＝pillar最高純度 ③コメディが物理の必然として内蔵 ④起承転結のビート。
- **未決ノブ**: 上昇は加速するか / 何人(合計体重)で止まるか / 天使自体に干渉できるか / 連れ去られた後の扱い。
- ※ 後述のアイデアカタログでも、この案を独立メインループへ昇格した「天使の綱引き」が**全案中スコア最高(88)**。

## 5. 議論中・未確定の論点

### 5.1 今朝のスケッチ案（洞窟・水位脱出）— 採否を模索中
- 洞窟の**最下層に転移**して開始 → **カート状の大容量容器**に宝/道具を詰めながら → **上昇する水位から逃げて**洞窟を脱出（垂直方向）。制限時間 **≤25分**。**ゴール（どこを目指すか）未定**。
- 付随ネタ `[※手書き解釈・要確認]`: 夜の洞窟 / 白い水晶を池に落として下層へ転移 / 水位が引くと安心・引くとマズい等の水位ギミック / 「6人で?」案（※SPECは2〜4人）/ ラグドールの "STUPID"（不安定さ）を活かす方針。
- **未解決の最大懸念**: 常時上昇する水位は「立ち止まる」ことを罰する。一方で天使救出は「立ち止まって複数人で引き止める」時間を要する → **2つの見せ場が構造的に衝突**。
  - 提案中の解: 水位を「常時上昇」でなく **段階的/クライマックス的**にする（探索フェーズ=平穏・救出可 → 終盤=水位サージで脱出大団円）。緊張と安心の混在も両見せ場も両立できる。
- 他懸念: カート×上り×水（浮力・滑り）の機構難度 / 25分の常時緊張は「安心の混在」を圧迫 / ゴール未定＝逃げる方向の可読性なし / 垂直×ラグドール×水位同期の実装難度。
- **プレッシャー源の捉え方（提案）**: コア＝「協力の物理ドタバタ」は不変。プレッシャー源（水位/敵/時間）は**交換可能なフレーム**。緩急(lull)を許すプレッシャー（反応的な敵/トラップ、フェーズ式サージ）が嗜好に合い、常時プレッシャー（常時上昇水位）はコアに不向き。→ SPECのBracken（反応的脅威）で moment-to-moment、水位は終盤クライマックスに**層化**する案。

### 5.2 コアの面白さ「一文」— 推敲中（まだ確定していない）
- エンジン: 「必死に支え合うほど、共倒れして笑う」。源は共有された物理の連動＝連帯責任。
- 候補:
  - A（推奨・中尺）「すべてが物理で繋がった世界で、重い宝と仲間を必死に支え合う——一人のよろめきが全員を巻き込み、その共倒れが笑いになる。」
  - B（凝縮）「繋がった仲間と必死に支え合うほど、共倒れして笑う。」
  - C（フル・哀愁込み）「重い宝も、倒れた仲間も、ときに身体そのものも物理で繋がっている。誰か一人の必死が全員の運命になり、その巻き込み事故が笑いに変わる協力ゲーム。」
- ※ 次セッションはここを詰めるのが自然な再開点の一つ。

## 6. アイデアカタログ（多角ブレストの成果）

> 16の発想レンズ × 並列生成 → 重複統合・クラスタ化した1巡目の成果（**11クラスタ / 43案**）。採点ショートリストと未踏領域、統合所見つき。

### 6.1 採点ショートリスト（最有望12件 / score 0-100）

**[88] 天使の綱引き（Soul Tug／救出綱引きをメインループに昇格）**
- フック: 死んだ脱力体を天使が上空へ吊り上げ、仲間が部位を掴んで地面へ引き戻す物理綱引きで蘇生。複数同時死亡で『誰を諦めるか』のトリアージ。FF誤射が直接の死因になり、救出中の仲間ごと天使に持っていかれる連帯事故が起きる。
- 有望な理由: 作者嗜好（死=ネタ、綱引き、連帯責任）に直撃し、既に固定制約／アイデアメモに『天使が連れ去る蘇生』が明記済みで動機を語れる。既存のgrab・質量シェア運搬・脱力ラグドールがそのまま部品になり実装現実性が突出して高い。洞窟抽出に追加するモジュールにも、単体ゲームにもなる二刀流。

**[85] カラダ橋／人体ブリッジ工法（自分の体が橋・梯子・足場になる自己建築）**
- フック: 資材が無いので胴体・腕を梁にして谷を渡る。渡る人数ぶん撓み、限界でgrabジョイントが弾けて橋ごと落下。最後の支え手をどう回収するかが毎ステージのパズル。踏まれた側がフニャっと潰れるFFが構造強度そのもの。
- 有望な理由: ラグドール＋grabのbreakForce／質量シェアという既存の物理が直接ゲームの強度になり、Gang Beasts/HFFの中間という手触り要件にも合致。クリップ映え（人間団子で誰の腕か分からない）が物理の必然から出る。運搬→自己建築という最もクリーンなコンセプト反転。

**[82] フリ子／ペンデュラム・パス（振り子で運動量を受け渡す垂直リレー）**
- フック: 天井のロープを掴んでブランコし、最高到達点で仲間をthrowで遠投、投げられた側が空中で次のフックをgrab。二重振り子の角速度×離すタイミングが操作で、勢い=資源。
- 有望な理由: 既存のgrab・throw・thrustが綺麗に運動量パスへ転用でき実装現実性が高い。物理由来のクリップ（あと一歩でフックに届かず救出される逆転）が量産される。重さ=負担を勢い=資源に反転させる発想が抽出案と直交。垂直/振動という運動軸の新しさ。

**[81] 息止め回廊（脱力＝死んだふりを攻めに使うステルス）**
- フック: 音に反応する盲目の徘徊者を、脱力ボタンで床に崩れ『死んだふり』して切り抜ける。立っている体をいつでも手放せるラグドール固有の攻略。運搬中は脱力できず最も危険。
- 有望な理由: ラグドールの脱力を欠点でなく能力に転じる逆転が秀逸で、緊張と笑いの混在という作者嗜好に合う。既存の脱力・質量シェア運搬がそのままリスク通貨（騒音）になり実装現実性が高い。死=ネタ化とも相性良。前進ではなく『動かない・音を出さない』逆ベクトルで抽出案と明確に別物。

**[80] 絡まりロープ錠（Knot Lock／常時連結の結び目トポロジー）**
- フック: 全員が1本のロープで腰を繋がれ、正しい巻き付き方・交差順を身体で作るのが鍵。1人が走ると全員引っ張られ意図せぬ絡まり。柔らかいラグドールでこそ毎回違う絡まり方をする。
- 有望な理由: 新規性が極めて高い（紐の結び目を多人数の身体で解く遊びは類例が少ない）。連帯責任が物理拘束で常時可視化され作者嗜好と一致。2-4人で難度が綺麗にスケール。リスクはネット同期下での多体絡まりの安定性で、検証次第だが核アイデアの独自性が高い。

**[79] 目隠しと操縦士（感覚非対称の物理コミュニケーション）**
- フック: 力持ちの実行役は視界を奪われ、目が見える指示役は非力で何も持てない。掴まれて引っ張られる感触・ぶつかった反動だけで意思疎通する。声＋身体接触で伝える。
- 有望な理由: ラグドールの『接触が姿勢に出る』性質を情報伝達系に変える唯一性が高い。実装は視界マスク＋既存grabで軽く、2人で完結。クリップ映え（右と言われ左へ全力で進む皮肉）も強い。協力の必然性が情報の非対称から構造的に生まれる。スコープが小さく縦切り検証に向く。

**[78] 巨人と小人（スケール非対称・投擲が移動手段）**
- フック: 1人だけ全長5mの鈍重な巨人、残りは手のひらサイズの俊敏な小人。巨人が小人を掴んで投擲し、小人が巨人には届かない精密操作を担う。質量比10倍の相互ロック。
- 有望な理由: 非対称の核を体格に置く明快さと、投擲＝移動・操作という反転が新規。grab/throwがそのまま主メカニクス。役割が固定的で説明しやすく2人からでも成立。リスクはラグドールの極端スケール差での物理安定性とカメラ設計で、実装難度はやや高めだが体験の独自性が魅力。

**[76] せーので押す王国（On The Count of Three／時間同期パズル）**
- フック: 全員のthrustを同フレーム帯に揃えないと作動しない超重量スイッチだけの城。物理動作の立ち上がり時間と個人差で『揃える』のが本当に難しく、0.3秒ズレると反力で全員吹き飛ぶ。
- 有望な理由: パズルの答えを空間でなく時間同期に置く新規性。失敗が物理的に派手な事故になり号令バラエティ的な笑いが出る。既存thrustと質量シェアで実装が軽い。協力の必然性が純化されている。リスクはネット遅延下での同時性判定で、許容窓の設計が肝になるが要素が少なく検証しやすい。

**[75] ヒトハシゴ／人間タワー（垂直登攀の統合）**
- フック: 仲間の体をアンカー・踏み台にして登る垂直シャフトレース。grabのbreakForceで支えきれず弾ける限界が物理で生まれ、踏まれた頭が手を離させて連鎖落下。
- 有望な理由: 質量シェア運搬を『水平→垂直』へ転用する自然な拡張で実装が読みやすい。崩落のクリップ映えが強く、2-4人で高さが綺麗にスケール。FF（踏みつけ）が崩壊トリガーで協力の必然性が高い。やや既視感（Gang Beasts的)はあるが土台が堅実で、抽出案にも組み込めるモジュール性。

**[74] 天秤都市／シーソー・シティ（重心そのものをパズル入力にする）**
- フック: 床・足場が全て支点で釣り合う天秤で、プレイヤーの体重と転がりだけが分銅。立ち位置=レベルデザイン。脱力体が傾いた面でズルズル滑り、踏ん張り続けるアクティブな格闘になる。
- 有望な理由: 物理量（質量・重心・トルク）を直接パズルにする知恵比べが新鮮で、ラグドールの重心の揺らぎが必須。連帯責任（1人の体勢崩れで全員の足場が傾く）が常時かかる。リスクは連結天秤のネット同期と『静的パズルで退屈化』しないかの設計で、ラグドールの滑落が動的要素を担保。

**[73] 電池人間と機械（出力供給の非対称・脱力をメカニクス中心に）**
- フック: 通電中だけラグドールのPIDゲインが立ち上がる機械役と、給電する電池役。電源が切れると全関節が脱力してぐにゃり崩れる。過充電で暴走自爆。
- 有望な理由: ラグドールの最も特徴的な状態（脱力）をゲームの中心リソースに据える発想が独自で、PID駆動という既存実装と概念的に直結し面接で語りやすい。協力の必然性（電池は持てず機械は給電なしで指一本動かない）が完全相互依存。リスクはゲインON/OFFの手触り調整で、要素が2役と少なく検証しやすい。

**[72] ヒューマンゴルフ大砲（Bodyflail Golf／仲間を打つ弾道スポーツ）**
- フック: 丸まった仲間を複数人で支える物理クラブで打ち出し、打たれた側が空中で腕を伸ばし壁を掴んで自己制御する。少ない総打数で全員をゴールへ。
- 有望な理由: throw実装の延長で対象が常に意思を持って空中制御する点が新鮮。クリップ映え（空中衝突→池ポチャ、打った本人だけガッツポーズ）が物理から必然。役割交代で相談が毎ホール生まれ協力の必然性も高い。やや競技ルール設計に工数がかかるが手触りの面白さが見込める。

### 6.2 全アイデア（11クラスタ・43案）

#### 人体を建材にする（橋・梯子・塔・ドミノ）— 縦/水平の自己建築
- **ヒトハシゴ／トルネード塔（垂直登攀の統合案）** — 仲間の体をアンカー・踏み台にして登る垂直シャフトレース。回転モーメント版(トルネード塔)はパンチで渦を生み遠心力で振り回す変奏。grabジョイントのbreakForceで『支えきれず弾ける』限界が物理で生まれる。
- **カラダ橋／人体ブリッジ工法（人間アーチ橋の統合案）** — 資材が無いので自分の胴体・腕を梁にして谷を渡す。渡る人数ぶん本当に撓み、限界でジョイントが弾けて橋ごと落下。最後の支え手をどう回収するかが毎回のパズル。デカ橋(隙間連続版)も同系。
- **トーテム・ホープス／脱力ジェンガ人塔／陣取りダルマ落とし（人間タワーの統合案）** — 1人では絶対届かない高さを仲間を積んで生成。プルプル揺れる土台が連鎖崩壊。バスケ得点版・PvP陣取り崩し版・耐久版に展開可能。
